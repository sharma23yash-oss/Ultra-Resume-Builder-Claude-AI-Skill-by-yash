# Interview Discovery — Branching Question Bank

## Overview

Conversational prompts to surface undocumented experiences. Ask naturally.
Branch based on answers. Every answer is a potential resume bullet.

**Goal:** Discover 3–6 new strong bullets the user hasn't documented yet.

---

## Opening Script

```
"Before I write your resume, I want to ask you a few questions.
Don't worry about formatting — just answer like you're telling a
colleague a story. I'll turn it into resume-ready language.

This usually uncovers experiences you haven't thought to include
that are directly relevant to [Company/Role].

Ready? I'll keep this to 7–10 questions max."
```

## CORE QUESTION BANK (Universal — All Roles)

### Q1: Impact Question
```
"Tell me about a time when something you did — at work, college, a project,
or even a side hustle — had a measurable impact. Could be money saved,
users gained, time cut, marks improved, anything with a number."

BRANCH A (if number mentioned):
→ "What exactly was the baseline and what did it change to?"
→ "Over what time period?"
→ "How many people were affected?"

BRANCH B (if no number mentioned):
→ "Can you estimate roughly? Even a ballpark helps — '30% faster' or
   'about 50 people used it'"
→ "Was there any recognition or feedback from someone above you?"

EXTRACT → Bullet: [Verb] + [What] + [Tool/Method] + [Metric]
```

### Q2: Problem-Solving Question
```
"What's the most complex problem you've had to solve — professionally or
academically — where you had to figure out the approach yourself?"

BRANCH A (technical problem):
→ "What tools or languages did you use?"
→ "Did you have to learn something new for this?"
→ "What was the alternative approach you rejected and why?"

BRANCH B (business/analytical problem):
→ "Who were the stakeholders involved?"
→ "What data did you work with?"
→ "What was your recommendation and what happened after?"

EXTRACT → Bullet showing analytical depth + independent judgment
```

### Q3: Leadership / Collaboration Question
```
"Have you ever led a group — formal or informal? Even organizing a club
event, managing a project team, or guiding juniors counts."

BRANCH A (formal team):
→ "How many people? Direct reports or peers?"
→ "What was the goal and did you hit it?"
→ "Any conflict you had to navigate?"

BRANCH B (informal):
→ "What was your specific role vs the group's role?"
→ "What decision did you make that the group followed?"

EXTRACT → Leadership bullet; flag for roles requiring team management
```

### Q4: Competition / Achievement Question
```
"Any competitions, hackathons, case studies, fests, or ranked exams
you've done? Shortlisted, participated, or won — all count."

BRANCH A (placed / won):
→ "What was the total pool? (e.g., 'top 20 of 400 teams')"
→ "What was the theme and what did you present or build?"
→ "Any judges, sponsors, or companies involved?"

BRANCH B (participated, no rank):
→ "What did you build or present?"
→ "What did you learn that you applied later?"

EXTRACT → Achievement line with team size ratio; frame as proof of skill
```

### Q5: Extracurricular / Activity Question
```
"What activities — clubs, volunteering, sports, arts, community work —
have you been involved in? Especially anything with a leadership angle."

BRANCH A (leadership in activity):
→ "Any events you organized? Approximate attendance or budget?"
→ "What was the outcome and who recognized it?"
→ "How does this skill connect to [target role]?" ← You frame it

BRANCH B (participation):
→ "What did you specifically contribute that others didn't?"
→ "Any collaboration with external organizations or sponsors?"

EXTRACT → Extracurricular bullet with JD-relevant framing
NOTE: Every extracurricular needs a 'why it matters for this job' hook
```

### Q6: Certification / Course Question
```
"Any online courses, certifications, or workshops you've done or are
currently doing? Even if incomplete — 'in progress' is fine."

BRANCH A (completed):
→ "When did you complete it? Do you have a certificate?"
→ "Did you apply anything from it in a project or work?"
→ "What platform? (Coursera, NISM, NSE, etc.)"

BRANCH B (in progress):
→ "What's your expected completion date?"
→ "I'll list it as '(In Progress)' — is that okay?"

EXTRACT → Certification with issuer + year + application context
```

### Q7: Startup / Freelance / Side Project Question
```
"Any side projects, freelance work, startups, or personal projects — even
early stage, even if they failed or are ongoing?"

BRANCH A (revenue / users):
→ "Any revenue, users, or other metrics?"
→ "What tools/stack did you use to build it?"
→ "Is there a GitHub repo or live URL I can include?"

BRANCH B (idea stage / prototype):
→ "What problem does it solve? How far along is it?"
→ "Any validation — survey, prototype, pilot users?"
→ "Can I list it as 'Founder' or 'Builder' with current status?"

EXTRACT → Project or Experience entry with GitHub/demo link
NOTE: Startup founder experience is real experience — frame it
```

### Q8: LinkedIn & GitHub Deep Dive
```
"Can you share your LinkedIn URL? I want to check if there are projects,
recommendations, or certifications there that aren't in your resume yet."

FOLLOW-UPS:
→ "I see you listed [X] on LinkedIn — tell me more about that"
→ "This recommendation from [Y] mentions [Z] — can I pull that in?"

"And your GitHub (if applicable)?"
FOLLOW-UPS:
→ "Which repos are you most proud of? Any with stars or forks?"
→ "Any contributions to others' open source projects?"
→ "Any deployed projects — live URLs?"

EXTRACT → Projects, open source contributions, showcased code skills
```

## ROLE-SPECIFIC QUESTION PACKS

### Finance / Banking / Investment Roles

```
F1: "Have you ever valued a company or built a financial model?
     Walk me through what inputs you used and what you concluded."
→ EXTRACT: DCF/valuation experience; tools used (Excel, Python, Bloomberg)

F2: "Do you track markets or have a personal portfolio? Even paper trading."
→ EXTRACT: Investment philosophy; portfolio metrics; sector knowledge

F3: "Any exposure to financial statements analysis — even for a case study?"
→ EXTRACT: Balance sheet, P&L, ratio analysis experience

F4: "Have you worked with any financial tools — Tally, Bloomberg, CMIE,
     ACE Equity, Excel for finance, Python for data?"
→ EXTRACT: Tools list for Skills section

F5: "Tell me about a company or sector you've researched deeply."
→ EXTRACT: Research skills; sector knowledge; report writing
```

### Marketing / Brand / Digital Roles

```
M1: "Have you run any campaigns — digital, social, event, or offline?
     What was the goal and what were the results?"
→ EXTRACT: Campaign metrics (CAC, CTR, ROAS, reach, conversions)

M2: "Any content you've created that got traction — posts, blogs, videos?"
→ EXTRACT: Content creation; engagement metrics; platform skills

M3: "Have you worked with tools like Google Analytics, HubSpot, Mailchimp,
     Meta Ads, Google Ads, or Canva?"
→ EXTRACT: MarTech stack for Skills section

M4: "Any market research — surveys, interviews, competitive analysis?"
→ EXTRACT: Research methodology; sample sizes; insights drawn
```

### Tech / Engineering / Product Roles

```
T1: "Walk me through your strongest project — architecture, tech stack,
     what problem it solved, and how many people used it."
→ EXTRACT: Project bullet with GitHub link + stack + metric

T2: "Any experience with APIs, databases, cloud services, CI/CD?"
→ EXTRACT: Technical depth; infrastructure skills

T3: "Have you contributed to any open source projects or helped others
     with their code on GitHub?"
→ EXTRACT: Open source contributions; community engagement

T4: "Any deployment experience — have you shipped something to production?"
→ EXTRACT: End-to-end delivery experience

T5: "Any experience with data — SQL queries, Excel analysis, Python scripts,
     dashboards?"
→ EXTRACT: Data skills; tools; analytical depth
```

## Discovery Session Management

### After Each Answer

1. **Confirm the metric:** "Just to confirm — was that [X] or [Y]?"
2. **Extract the bullet draft:** "Here's how I'd write that: [bullet]. Does that sound accurate?"
3. **Tag and move on:** Don't dwell — target 8 questions max

### When to Stop

Stop when:
- You've covered all critical JD gaps
- User has answered 8–10 questions
- No new information is emerging

### End of Discovery

```
"Great — I've uncovered [N] new experiences you hadn't documented before.
Here's what I found:

[List discovered bullets]

I'll weave these into your resume. The ones marked (★) are strong matches
for the JD and will likely improve your ATS score significantly.

Ready to see the full resume?"
```
